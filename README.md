# Werewolves Revenge — Website

The companion website for [Werewolves Revenge](https://store.steampowered.com/app/1867320/Werewolves_Revenge/), an immersive social deduction game for 8–16 players. Built with [Astro](https://astro.build) and [Tailwind CSS](https://tailwindcss.com).

The site covers:

- **Home / Play** (`/play`) — Landing page with the game pitch, faction showcase, gameplay loop, and Steam link.
- **Role Directory** (`/roles`) — a searchable, filterable grid of every role in the game, with portrait artwork.
- **Role Details** (`/roles/[id]`) — a dedicated page per role with full artwork, faction/class tags, and description.

## 🎨 Faction & class colors

Faction and class colors are centralized in [`src/data/factions.ts`](./src/data/factions.ts) and [`src/data/classes.ts`](./src/data/classes.ts) as plain hex codes — no Tailwind utility classes to manage. Each page sets a `--faction` / `--class` CSS custom property inline, and the shared rules in [`src/styles/global.css`](./src/styles/global.css) (`.faction-text`, `.faction-card`, `.class-border`, etc.) consume it. To change a color, edit the hex value in one of those two files; it updates everywhere automatically.

## 📦 Retrieving role data

`src/data/roles.json` and the artwork in `public/images/roles/` are generated by running [`src/export_roles.py`](./src/export_roles.py) inside the Unreal Editor's Python console. It pulls every role from the allowed sets, exports each role's image/portrait textures as PNGs, and writes the combined role list to `roles.json` (already pointing at the `.webp` filenames the next step produces).

Unreal's exporter only writes PNGs, so after running the export, optimize them for the web:

```sh
npm run optimize-images
```

This converts every PNG in `public/images/roles/` to WebP (capped at 1000px on the longest edge, quality 82) and deletes the source PNG once its `.webp` replacement is written — cutting the role art from ~1.1GB down to ~40MB with no visible quality loss. It's safe to re-run after every export: `export_roles.py` skips re-exporting any texture that already has a `.png` or `.webp` on disk.
